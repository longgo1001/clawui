## 飞书图片发送问题排查 (09:33-10:07)
- **问题**: Agent 回复中的图片无法发送到飞书
- **根因**: 
  1. Agent 用 `read` 工具查看图片只在 agent 端渲染，不会发到飞书
  2. `normalizePossibleLocalImagePath` 要求消息是纯路径（无空格/其他文本），agent 回复通常包含上下文文字
  3. `/tmp` 不在 `mediaLocalRoots` 允许列表中 → `LocalMediaAccessError: path-not-allowed`
- **解决**: 使用 `openclaw message send --channel feishu --account main --target <user_id> --media <workspace_path> -m "caption"` 直接发送
- **允许的媒体目录**: `~/.openclaw/{media,agents,workspace,sandboxes}` + OpenClaw 专用 tmp 目录
- **教训**: 向飞书发图片时，图片必须放在 workspace 目录下，然后用 `openclaw message send` CLI 发送
